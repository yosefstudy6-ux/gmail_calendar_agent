# L06 Gmail Calendar Agent

פרויקט סיום למטלה 6 בקורס סוכני בינה מלאכותית (AI Agents).

הפרויקט מממש סוכן אוטומטי המשלב בין Gmail, Google Calendar ויכולות ניתוח שפה טבעית מבוססות Google Gemini, לצורך זיהוי בקשות לפגישה מתוך מיילים וביצוע פעולה מתאימה באופן אוטומטי.

## תיאור המערכת

המערכת מבצעת את תהליך העבודה הבא:

1. קוראת הודעות רלוונטיות מתוך Gmail.
2. מחלצת את נושא ההודעה, השולח ותוכן המייל.
3. מנתחת את תוכן ההודעה באמצעות רכיב LLM כדי לזהות האם מדובר בבקשת פגישה.
4. אם זוהתה פגישה, מחלצת פרטים רלוונטיים כגון תאריך, שעה, משתתפים, מיקום וסיכום.
5. בודקת זמינות ביומן Google Calendar.
6. אם הזמן פנוי, יוצרת אירוע ביומן.
7. אם הזמן תפוס, שולחת מייל תשובה אוטומטי לשולח.
8. מסמנת את ההודעה כמעובדת כדי למנוע טיפול חוזר.

## יכולות עיקריות

- זיהוי מיילים של בקשות לפגישה.
- ניתוח שפה טבעית בעזרת Google Gemini.
- בדיקת זמינות ב-Google Calendar.
- יצירה אוטומטית של אירועים ביומן.
- שליחת תשובת סירוב אוטומטית כאשר המועד תפוס.
- סימון הודעות שטופלו באמצעות Gmail Label.

## טכנולוגיות וכלים

- Python 3.10+
- `uv` לניהול תלויות והרצה
- Google Gemini לצורך ניתוח שפה טבעית

- Google Calendar API
- OAuth 2.0
- Environment Variables לניהול מפתחות רגישים
- GitHub לצורך ניהול קוד והגשה

## קבצים מרכזיים

- `main.py`  
  קובץ ההרצה הראשי. אחראי על ניהול הזרימה המלאה של הסוכן.

- `src/gmail_service.py`  
  מטפל בשליפת הודעות מ-Gmail, חילוץ פרטי מייל, שליחת תשובה וסימון הודעות כמעובדות.

- `src/calendar_service.py`  
  מטפל בבדיקת זמינות ביומן וביצירת אירועים.

- `src/meeting_parser.py`  
  מנתח את תוכן ההודעה ומחזיר מבנה נתונים המתאר האם מדובר בבקשת פגישה ומהם הפרטים שחולצו.

- `src/config.py`  
  מכיל הגדרות מערכת, שמות labels, scopes, נתיבי קבצים והגדרות נוספות.

## דרישות מוקדמות

לפני הרצה יש לוודא:

- Python 3.10 או גרסה חדשה יותר מותקנים.
- `uv` מותקן במערכת.
- `credentials.json` קיים מקומית בתיקיית הפרויקט או בנתיב שהוגדר בקובץ `config.py`.
- Gmail API ו-Google Calendar API מופעלים בפרויקט Google Cloud.
- המשתמש שהריץ את המערכת מוגדר כ-Test User אם האפליקציה עדיין במצב Testing.
- מפתח Gemini מוגדר כמשתנה סביבה.

## התקנה והרצה

### 1. שכפול המאגר

```powershell
git clone <REPOSITORY_URL>
cd l06_agent
```

### 2. התקנת תלויות

```powershell
uv sync
```

פקודה זו מתקינה אוטומטית את כל הספריות המוגדרות ב-`pyproject.toml`.

### 3. הגדרת משתנה סביבה עבור Gemini

ב-Windows PowerShell:

```powershell
$env:GEMINI_API_KEY="YOUR_GEMINI_API_KEY"
```

יש לוודא ששם משתנה הסביבה תואם בדיוק למה שהקוד מצפה לו.

### 4. הוספת קובץ OAuth

יש לשים את הקובץ `credentials.json` במיקום המתאים לפי ההגדרות של `config.py`.

חשוב:
- אין להעלות את `credentials.json` ל-GitHub.
- אין להעלות את `token.json` ל-GitHub.
- אין להעלות מפתחות API אמיתיים.

### 5. הרצת הפרויקט

```powershell
uv run main.py
```

בהרצה הראשונה ייפתח חלון דפדפן לצורך אימות Google OAuth. לאחר ההתחברות ייווצר `token.json` באופן אוטומטי.

## דוגמת זרימת עבודה

כאשר מתקבלת הודעה עם בקשת פגישה:

- אם לא מדובר בבקשת פגישה, ההודעה תסומן כמעובדת.
- אם זוהתה בקשת פגישה אך חסרים תאריך או שעה, לא ייווצר אירוע.
- אם זוהתה בקשת פגישה והמועד פנוי, ייווצר אירוע ב-Google Calendar.
- אם זוהתה בקשת פגישה והמועד תפוס, יישלח מייל תשובה אוטומטי לשולח.

## סימון הודעות כמעובדות

אם התווית אינה קיימת, המערכת יוצרת אותה אוטומטית בזמן הריצה.
